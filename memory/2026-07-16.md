## Permissions Race Condition Fix
- Implemented a robust retry mechanism (up to 3 attempts, 500ms delay) in `createUserDocument` in `src/firestore.js` to catch temporary `permission-denied` errors. This handles instances immediately after user signup where the newly authenticated client state is propagating to Firestore.
- Verified successful builds using `bun run build`.

## Unauthenticated Read for Username Availability Check & Signup Logging
- Identified that `isUsernameUnique` checks executed before Firebase account creation, causing unauthenticated reads on `users` collection to fail against the `allow read: if isOwner(uid) || isAdmin()` Firestore rule.
- Split `read` rule in `firestore.rules` under `users/{uid}` into a private `get` rule (owner/admin only) and an open `list` query rule with filter limit check: `allow list: if request.query.limit == 1;`.
- Updated `isUsernameUnique` query in `src/firestore.js` to enforce `limit(1)`, matching the rule criteria.
- Added console logging before/after `validateUsername`, `isUsernameUnique`, `createUserWithEmailAndPassword`, and `createUserDocument` in `auth/auth.js` to instrument the sign-up process.
- Verified successful builds using `bun run build`.
